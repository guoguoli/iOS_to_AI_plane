# 2026-04-21

## 修复 test_async.py 中的异步客户端兼容问题

- **问题**：`AsyncClient(app=app, base_url="http://test")` 在 httpx 0.20+ 中已废弃 `app` 参数，当前环境 httpx==0.28.1 报 `TypeError: AsyncClient.__init__() got an unexpected keyword argument 'app'`。
- **修复**：改用 `ASGITransport(app=app)` + `AsyncClient(transport=transport, base_url="http://test")`。
- **结果**：3 个测试全部通过（Python 3.13.9，pytest-asyncio 1.3.0，httpx 0.28.1）。

## 修复 ios_to_ai_plane conda 环境中的循环导入问题

- **现象**：在 `conda run -n ios_to_ai_plane` 下运行报 `ImportError: cannot import name 'TestClient' from partially initialized module 'fastapi.testclient' (circular import)`。
- **根因**：httpx 安装包的 `_client.py` 源文件被错误覆盖为 `test_async.py` 的内容（`/opt/anaconda3/envs/ios_to_ai_plane/lib/python3.14/site-packages/httpx/_client.py`），导致 httpx → fastapi.testclient → starlette.testclient → httpx 循环依赖。
- **修复**：`pip install --force-reinstall httpx` 重新安装 httpx，恢复原始源码；同时将 starlette 从 1.0.0 降级到 0.52.1。
- **结果**：3 个测试全部通过（Python 3.14.3，pytest 9.0.2，httpx 0.28.1，starlette 0.52.1）。

## 修复 test_database.py 中的导入错误

- **错误 1**：`NameError: name 'pytest' is not defined`——第 106 行使用 `@pytest.fixture` 但未导入 pytest。**修复**：添加 `import pytest`。
- **错误 2**：`MovedIn20Warning: declarative_base()`——`from sqlalchemy.ext.declarative import declarative_base` 在 SQLAlchemy 2.0+ 已弃用。**修复**：改为 `from sqlalchemy.orm import declarative_base`。
- **结果**：2 个测试全部通过（Python 3.14.3，conda env ios_to_ai_plane）。
