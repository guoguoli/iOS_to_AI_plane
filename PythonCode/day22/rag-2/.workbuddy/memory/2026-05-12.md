# 2026-05-12 工作日志

## 修复 use_ chroma.py 报错问题

**环境**：conda `ios_to_ai_plane`，chromadb 新版

### 问题汇总及修复方案

1. **弃用 API `Settings(chroma_db_impl=...)`**
   - 原因：新版 chromadb 已完全废弃旧版 Settings 配置
   - 修复：将 `chromadb.Client(Settings(...))` 改为 `chromadb.EphemeralClient()`

2. **`collection_exists()` 方法不存在**
   - 修复：改为 `[c.name for c in chroma_client.list_collections()]` 判断

3. **`metadata` 中不支持 `hnsw:M` / `hnsw:ef_construction`**
   - 新版只支持 `hnsw:space` 在 metadata 中，其他 HNSW 参数已移出
   - 修复：删除 metadata 中不支持的 HNSW 参数

4. **自动 embedding 下载超时**（`all-MiniLM-L6-v2` 79.3MB 从 hugging face 下载）
   - 修复：`add/update/query` 时直接传 `embeddings=[...]` / `query_embeddings=[...]`，绕过自动下载
   - 生产环境应接入 DashScope `qwen_embed_function`

5. **批量 add 时 ids 与 documents 数量不匹配**
   - 原来 ids 生成100个，但 documents 只有3条
   - 修复：ids 与 documents 数量对齐

**最终结果**：脚本完整运行通过（退出码 0）
